Tutorial

Prepare a page for translation

This tutorial walks you through making your first page ready for the MediaWiki Translate extension using TranslateTagger. By the end you will have wikitext with proper <translate> tags that editors can immediately start translating.

Before you start You need a wiki page whose raw wikitext you can copy. You do not need any account or software — only a browser.

Step 1 — Copy the raw wikitext

Open the wiki page you want to prepare. Click Edit (or Edit source), select all the wikitext in the editor, and copy it.

Step 2 — Open TranslateTagger

Visit translatetagger.toolforge.org. You will see two text areas: Input on the left and Output on the right.

Step 3 — Paste your wikitext

Click inside the left textarea and paste (Ctrl+V or Cmd+V) the wikitext you copied.

Step 4 — Convert

Click the Convert to Translatable Wikitext button. The right panel fills with your wikitext wrapped in <translate> tags.

Step 5 — Review the output

Scan the output. Links will have become [[Special:MyLanguage/…]], and dynamic values will be wrapped in <tvar> tags. The translatable prose appears inside <translate>…</translate> blocks.

Input wikitext
Welcome to the [[Wikimedia|Wikimedia Foundation]]!
Visit [https://wikimediafoundation.org our site].
Output (translatable)
<translate>Welcome to the [[Special:MyLanguage/Wikimedia|<tvar name="1">Wikimedia Foundation</tvar>]]!
Visit [<tvar name="2">https://wikimediafoundation.org</tvar> our site].</translate>

Step 6 — Copy and paste back into the wiki

Click Copy to Clipboard. Go back to your wiki editor, select all, and paste the converted wikitext. Save the page, then use Mark this page for translation to activate the Translate extension.

You're done! The page is now ready for community translators. Each <translate> block becomes a separate translation unit in the Translate extension.
How-to Guide

Convert an existing page to a translatable format

Use this guide when a page already exists and you need to make it translatable without altering its visible content.

  1. Open the page and go to Edit source.
  2. Select all the wikitext and copy it.
  3. Open TranslateTagger and paste into the input area.
  4. Click Convert to Translatable Wikitext.
  5. Click Copy to Clipboard.
  6. Return to the wiki editor, replace the entire content with the copied output, and save.
  7. On the page, click Mark this page for translation to activate the Translate extension.
Note on existing translate tags If the page already contains <translate> tags, TranslateTagger will skip conversion and return the input unchanged. Remove any existing tags first if you need a full re-conversion.
How-to Guide

Handle complex wikitext safely

TranslateTagger handles all standard wikitext structures automatically. You do not need to simplify the wikitext before conversion.

Supported complex structures

StructureHow it is handled
Templates {{…}}Preserved verbatim inside <tvar> tags — not wrapped for translation.
Tables {|…|}Cell content is wrapped; table markup and attributes are preserved.
Internal links [[Page|Label]]Page name becomes Special:MyLanguage/Page; label is translatable.
External links [URL text]URL is placed in a <tvar>; anchor text is translatable.
Code / syntaxhighlightContent of code blocks is preserved and not translated.
Section headings ==Title==Entire heading line is wrapped in a single <translate> block.
List items * # ; :List marker is preserved; item text is wrapped for translation.
File / image embedsFile name placed in <tvar>; caption and alt text are translatable.
How-to Guide

Use the JSON API

Automate conversion in scripts or bots by calling the /api/convert endpoint.

Send a conversion request

curl -X POST https://translatetagger.toolforge.org/api/convert \
  -H "Content-Type: application/json" \
  -d '{"wikitext": "Hello [[World|world]]."}'

Example response

{
  "converted_text": "<translate>Hello [[Special:MyLanguage/World|world]].</translate>"
}

Python example

import requests

response = requests.post(
    "https://translatetagger.toolforge.org/api/convert",
    json={"wikitext": "Hello [[World|world]]."}
)
print(response.json()["converted_text"])
Rate limits The tool runs on Wikimedia Toolforge. Respect standard Toolforge usage policies and avoid sending large batches in tight loops.
How-to Guide

Use as a UserScript (in-wiki)

You can run TranslateTagger directly inside the MediaWiki editor via a gadget/UserScript, without leaving the wiki.

  1. Open your common.js page (e.g., Special:MyPage/common.js).
  2. Add the following import line and save: 
    mw.loader.load('//www.mediawiki.org/w/index.php?title=MediaWiki:Gadget-TranslateTagger.js&action=raw&ctype=text/javascript');
  3. Open any page for editing — a TranslateTagger button will appear in the toolbar.
  4. Click it to convert the editor contents in place.

Explanation

Why TranslateTagger exists

The MediaWiki Translate extension requires pages to be explicitly marked up with <translate> tags before they can be translated. Doing this by hand is:

  • Slow — every paragraph, heading, and list item needs individual attention.
  • Error-prone — a single misplaced tag can break the entire page structure.
  • Inaccessible — volunteer editors unfamiliar with the tag syntax are blocked from contributing.

TranslateTagger automates the tagging step so that editors can focus on content and translation coordination, not markup mechanics. It is especially useful for recurring Wikimedia events (Wikimania, Hackathons) where dozens of pages need preparation on a short timeline.

Explanation

How it works

TranslateTagger uses a two-pass, cursor-based tokenizer to process wikitext:

Pass 1 — Tokenization

The input string is scanned character-by-character. When a known delimiter is encountered (e.g., [[, {{, ==, <), a specialized handler function is called to consume that element and return a transformed token. The cursor advances past the consumed characters.

Pass 2 — Merging

Adjacent translatable text tokens are merged into a single <translate> block. Non-translatable tokens (templates, code blocks, math) are kept as-is between translation blocks.

Explanation

What are tvar tags and why do they matter?

<tvar name="n">…</tvar> tags mark non-translatable variables inside a translation unit. They serve two purposes:

1. Protect dynamic or structural content

URLs, template syntax, and file names must not be altered by translators. Wrapping them in <tvar> hides them from the translation interface while keeping them in the final rendered output.

2. Allow translators to reorder content

A translator working in a language with different word order can move the placeholder token (e.g., $1) to the correct position without touching the underlying URL or template call.

TranslateTagger assigns each <tvar> a unique sequential numeric name (1, 2, …) scoped to the entire document, ensuring there are no duplicates across sections.
